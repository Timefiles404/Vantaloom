# Vantaloom

Vantaloom 是一个面向真实工程项目的 AI base app。它的目标不是生成一次性的玩具代码，而是让用户通过自然语言“编织”出可以持续演进、验证、发布和维护的稳定工程项目。

项目仍处在基础设施搭建阶段。当前仓库只包含技术选型、项目骨架、发布链路规划和工程边界文档，暂不包含实际产品功能。

## 技术栈

Vantaloom 当前采用以下主栈：

- 桌面壳：Wails v2
- 本地后端：Go
- 前端：React、TypeScript、Vite
- 本地状态：SQLite，后续以 WAL 模式承载项目状态、运行日志和恢复点
- 工程验证：GitHub Actions
- 发行渠道：GitHub Releases
- 自更新：基于签名发布清单的应用内更新机制

这个组合优先考虑稳定性、可维护性、较低内存占用、较快编译速度和良好的前端开发体验。

## 产品方向

Vantaloom 希望成为一个本地桌面工程编排器。用户描述意图后，应用会把需求拆成可跟踪的计划、任务、代码变更、CI 验证、修复循环和发布产物。

长期特性方向包括：

- 自然语言驱动的工程项目创建与演进
- 可恢复的 agent 运行时
- Git 分支、提交和检查点管理
- GitHub CI 作为权威验证环境
- 长文本、日志、diff 和计划视图的稳定展示
- 应用自更新和快速迭代
- 面向开源社区和商业团队的双许可证模式

## 开源与商业

Vantaloom 计划采用开源加商业的双许可证模式：

- 社区版本：AGPL-3.0-or-later
- 商业授权：用于闭源集成、私有改造、白标分发和企业支持

品牌名称、标识和相关商标不随源码许可证授权。
