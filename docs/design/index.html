UI 素材与设计准则平铺整理

这里不是产品页面，只是把当前 UI 库的风格方向、素材来源、组件取向和注意事项放在一个浏览器里快速扫一遍。

图标系统 素材来源 已拉取本地

Nucleo

  • 适合作为后续完整图标体系候选。
  • 分类覆盖工程工具常用区域：文件、图表、设备、开发、用户、安全、布局等。
  • 支持 Outline、Fill、Colored 等风格，以及 SVG、TSX、React、PNG 等导出。
  • 已本地保存公开 MIT 包 nucleo-glass-icons，包含 101 个 React 图标组件。

本地目录：assets/icons/nucleo-glass

毛玻璃 黑色卡片 MIT 来源记录

Uiverse 黑色毛玻璃卡片

  • 参考黑色卡片、半透明边框、轻微层次和玻璃质感。
  • 适合抽象成 panel、modal、agent card、status card 的材质 token。
  • 不直接复制页面，只提炼为 solid 与 glass 两种材质模式。
  • 长文本区域要克制玻璃效果，避免阅读疲劳。

当前环境被 Cloudflare 拦截，已保留来源与授权记录，未自动拉取源码。

动效素材 微交互 已拉取本地

React Bits

  • 适合收集文字动效、状态反馈、背景动效和交互动效。
  • Vantaloom 只把它作为动效素材池，不作为主视觉。
  • 优先选择短、轻、可关闭的动效。
  • 长日志、diff、计划内容不使用持续动画。

本地目录：assets/motion/reactbits，已保存 131 个组件族的 registry JSON。

UX 框架 组件体系 已拉取本地

shadcn/ui

  • 借鉴开放代码式组件库思路，而不是依赖难改的黑盒 UI 库。
  • 使用 CSS variables 和语义 token 管理主题。
  • 优先组件：Button、Input、Dialog、Tooltip、Tabs、Sidebar、Command、Scroll Area、Resizable。
  • 后续组件代码应进入 packages/ui，由应用按需组合。

本地目录：assets/ux/shadcn，已保存 16 个候选组件 registry JSON。

Vantaloom UI 黑白切换 纯色与毛玻璃

当前设计准则

  • UI 库先承载 token、主题、材质和工具函数，不负责拼页面。
  • 视觉以黑白可切换、纯色面板和毛玻璃面板为主。
  • 工程工具内容优先，动效只用于反馈、状态转场和少量品牌区域。
  • 所有核心 UI 基础能力放入 packages/ui，避免只服务桌面端。
  • 后续长文本、日志、diff、agent trace 需要稳定尺寸和虚拟滚动策略。
