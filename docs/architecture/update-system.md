# 自更新系统

Vantaloom 计划从一开始依赖 GitHub Releases 发行，并通过应用内更新机制获取新版本。

## 目标

- 不依赖本地机器发行正式版本
- 由 GitHub Actions 构建多平台产物
- 由发布清单描述版本、平台、下载地址、摘要和签名
- 应用更新前必须校验产物完整性
- 更新失败时保留回滚空间

## 清单字段预留

```json
{
  "version": "0.0.0",
  "notes": "",
  "pub_date": "",
  "platforms": {}
}
```

正式实现前需要补充签名算法、密钥管理、回滚策略和平台差异。
