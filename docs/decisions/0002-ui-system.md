# 0002 建立 Vantaloom UI 库框架

## 状态

已接受

## 背景

Vantaloom 后续会同时拥有桌面端、网页端和可能的移动端入口。UI 能力不应只散落在 `apps/desktop`，否则远程开发和 Web 扩展会重复实现视觉与交互基础。

## 决策

新增 `packages/ui` 作为 UI 库基础包，先承载设计 token、主题 CSS、材质模式和工具函数。

视觉方向采用可切换黑白主题，以纯色和毛玻璃为主要材质。UX 与组件组织借鉴 shadcn/ui 的开放代码、组合式和语义 token 思路。动效选材参考 React Bits，但只作为微交互和状态反馈素材，不作为主视觉负担。

## 影响

- 桌面端不会独占 UI 基础能力
- 后续 Web 和移动端可以复用设计语言
- 组件可以按能力逐步加入，而不是一次性拼装页面
- 引入具体第三方素材前需要记录授权、依赖和性能影响
