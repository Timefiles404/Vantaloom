# 工程规范

这些规则用于约束 Vantaloom 后续演进，不放在公开 README 中。

## 结构

- 代码按能力边界放入目录，不把源码平铺在单层目录中
- 一个文件只承载一个清晰责任
- 当两个板块可以独立变化时，优先拆成两个文件
- 避免文件膨胀，接近 300 行时主动评估拆分
- 不让应用层、agent 层、协议层和工具层相互穿透

## 变更

- 提交信息尽量使用中文
- 每次变更范围保持清晰
- 不做无关重构
- 不提交本地密钥、令牌和机器特定配置

## 验证

- GitHub CI 是正式验证入口
- 本地验证用于快速反馈，不作为发行依据
- 失败日志应沉淀为可解释、可复现的问题
